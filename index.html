Client-Server-Netzwerk Machen tipologie Ist ein dezentrales Netzwerk, das keinen verbindlichen Strukturen unterliegt. Allerdings sind alle Netzwerk-Stationen irgendwie miteinander verbunden. Beim Ausfall einer Verbindung gibt es im Regelfall einige alterative Strecken, um den Datenverkehr unterbrechungsfrei fortzuführen. Abhängig davon, ob alle Computer im Netz untereinander verbunden sind oder nur ein Teil, wird zwischen Vollvermaschung und Teilvermaschung unterschieden. Bei der Vollvermaschung ist jeder Computer mit allen anderen verbunden. Durch die alternativen Wege bietet die Maschen-Topologie eine hohe Ausfallsicherheit, allerdings ist der Verkabelungsaufwand sehr hoch. Bestes Beispiel für diese Topologie ist das Internet.
Client-Server-Netzwerk Bei der Ring-Topologie sind die Netzwerkstationen in einem Kreis angeordnet. Die Information, die übertragen werden soll, wird von Teilnehmer zu Teilnehmer weitergeleitet, bis sie ihren Bestimmungsort erreicht. Bei dieser Topologie wird regelmäßig ein sogenannter Token (= bestimmtes Bit-Muster) durch das Netzwerk geschickt. Alle Daten werden in eine Richtung übertragen, in einem sogenannten uni-direktionalen Broadcast. Der Computer, der gerade im Besitz des Tokens ist, darf Daten senden. Um eine neue Maschine an den Ring anzuschließen, unterbricht man den Ring, fügt die neue Maschine ein und schließt den Ring wieder zusammen. Größere Netzwerke verwenden beim Einfügen eines neuen Computers ein spezielles Verbindungsgerät (z.B. Switch oder Router), an das alle Maschinen angeschlossen werden, um den Ring nicht zu unterbrechen. Damit das Netzwerk funktioniert, müssen alle Maschinen angeschlossen sein. Wird eine Maschine abgeschlossen, oder ist sie defekt, wird der Datenfluss unterbrochen.
Client-Server-Netzwerk Ein P2P-Netzwerk wird in der Regel für zuhause oder in kleinen Unternehmen eingesetzt. Bei diesem Netzwerkmodell kommunizieren die Rechner direkt miteinander. Ein P2P-Netzwerk wird auch als Arbeitsgruppe bezeichnet und wird für Umgebungen eingesetzt, in denen nicht mehr als 10 Rechner miteinander verbunden sind. Die einzelnen Rechner, in einer solchen Arbeitsgruppe werden als Peers (=Gleichrangige) bezeichnet. In einem P2P-Netzwerk gibt es nämlich keinen Server zur zentralen Verwaltung der Netzwerkressourcen. Möglichkeiten in einem P2P-Netzwerk: ▪ Jeder Rechner bestimmt selbst, welche Ressourcen er im Netzwerk den anderen Netzwerkteilnehmern zur Verfügung stellt. ▪ Peripheriegeräte (Drucker, Scanner, …) können gemeinsam genutzt werden. ▪ Daten können ohne zusätzliche Speichermedien gemeinsam genutzt werden. Vorteile: ▪ Alle Computer sind gleichberechtigt. ▪ Ist kostengünstiger als Servernetzwerke (kein leistungsstarker Server notwendig). ▪ Keine spezielle Netzwerksoftware erforderlich. ▪ Benutzer verwalten sich selbst. Kein Administrator nötig. ▪ Keine hierarchische Netzwerkstruktur. ▪ Kann schnell aufgebaut werden. ▪ Aufgrund der beschränkten Größe überschaubar. Nachteile: ▪ Wenige Benutzer bzw. wenige Workstations sind möglich; max. 10 ▪ Keine zentralen Sicherheitsaspekte. ▪ Beschränkte Ausbaumöglichkeiten (=Skalierbarkeit). ▪ Keine gemeinsame Verwaltung: Hoher Koordinierungsaufwand. ▪ Für eine optimale Ressourcennutzung müssen immer alle PCs eingeschaltet sein (Datenfreigabe bzw. Drucker). ▪ Die Administration und Datensicherung muss von jedem Benutzer selbst vorgenommen werden (keine zentrale Datenverwaltung bzw. -sicherung) ▪ Wenige Steuerungsmöglichkeiten

Client-Server-Netzwerk Größere Unternehmen mit komplexen Aufgabenstellungen verwenden Client-ServerNetzwerke. Beim Client-Server-Netzwerk wird die Bearbeitung der Prozesse auf die einzelnen Arbeitsstationen und einen zentralen Server aufgeteilt. Auf dem Server werden die Dateien der Benutzer zentral gespeichert und der Zugriff auf die Netzwerkressourcen, wie z.B. Drucker, Datenlaufwerke oder Software, gesteuert. Zur Verwaltung von Client-Server-Netzwerken sind Administratoren notwendig. Ihre Aufgabe ist es, den Netzwerkbetrieb zu verwalten und die Aufrechterhaltung der Netzwerkinfrastruktur zu gewährleisten. Im Vergleich zu P2P-Netzwerken hat man beim Client-Server-Modell Möglichkeiten Sicherheitsstrategien für die Datensicherung und die Zugriffsteuerung (Benutzerechte) zentral festzuglegen. Vorteile: Zentralisierte und strukturierte Benutzer- und Ressourcenverwaltung Zentrale Rechtevergabe (Zugriffkontrolle für die Daten) Zentrales zur Verfügung stellen von den Netzwerkressourcen ▪ Unterscheidung zwischen Benutzern möglich (nicht alle Benutzer sind gleichrangig) ▪ Ausbaufähigkeit (nach oben offenes Netzwerkwachstum) ▪ Zentrale Datenbestände erlauben effiziente Backup-Strategien (z.B. RAID, …) ▪ Ausfallsicherheit (Redundanz der Daten und Serverdienste) ▪ Softwareaktualisierungen bzw. Sicherheitsupdates müssen nur am Server heruntergeladen werden und werden dann an die Clients verteilt→Clients bekommen von Updates wenig mit ▪ Einheitliche Arbeitsplatzrechner→Nutzer hat auf jeden Rechner Zugriff auf seine Daten im Netzwerk ▪ Löschen und Hinzufügen von Ressourcen ohne Störung des laufenden Betriebs Nachteile: ▪ Höhere Anschaffungskosten für Serverhardware und spezialisierte Serversoftware (Serverbetriebssystem, Sicherungssoftware, …) ▪ Hoher Anfangs(Initial-)aufwand bei der Ersteinrichtung ▪ Server als schwaches Glied – Ausfall des gesamten Netzwerkes bei gezieltem Angriff ▪ Server muss gegen Ausfall und Datenverlust gesichert werden → Doppelte Datenhaltung auf mehreren Servern, bei Ausfall soll ein Ersatzserver einspringen ▪ Server muss hohe Fehlertoleranz aufweisen ▪ Der Server muss immer in Betrieb sein, damit Clients auf die Daten zugreifen können ▪ Hohe Ausbildungs- und Schulungskosten für Administratoren (laufende Personalkosten) bzw.
